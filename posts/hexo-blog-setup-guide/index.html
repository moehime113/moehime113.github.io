Hexo 博客建站实录

折腾第一步：让博客先跑起来

建站就像搭积木，环境弄好，剩下就是顺水推舟。

1. 搞定基础环境

备好两样工具：

  • Git & GitHub：版本控制 + GitHub Pages 托管
  • Node.js：Hexo 运行基础，官网下载稳定版安装即可

2. 初始化 Hexo 与主题

# 安装 hexo-cli，初始化博客目录
npm install -g hexo-cli
mkdir your_blog && cd your_blog
hexo init && npm install

安装 Butterfly 主题：

npm install hexo-theme-butterfly

修改根目录 _config.yml，将 theme 设为 butterfly。执行 hexo s，打开 http://localhost:4000 即可预览。

部署上线：让世界看到你的博客

1. 部署到 GitHub Pages

安装部署插件：

npm install hexo-deployer-git

在 _config.yml 中配置 GitHub 仓库地址和分支（推荐 main），然后执行：

hexo clean && hexo g && hexo d

2. 进阶玩法：Vercel 托管

如果想要更快的国内访问速度，可用 Vercel 托管静态页面。Vercel 支持自定义域名，适合作为 GitHub Pages 的补充方案。

魔改与避坑指南

跟着教程走一帆风顺，自己动手加特效（轮播图、Aplayer 音乐）时，玄学问题就来了。

❌ 坑一：静态资源加载失败

现象：本地正常，部署后控制台一片红 ERR_ABORTED，排版错乱、图标变方块。

原因：部分国内 CDN 服务不稳定或已停运，在 GitHub Pages / Vercel 环境可能被拦截。

✅ 解决：在 _config.butterfly.yml 中将第三方资源替换为可靠的公共 CDN，如 unpkg.com 或 cdn.jsdelivr.net/npm/。

❌ 坑二：GitHub Actions 自动部署后页面不更新

现象：Workflow 显示构建成功，但网页内容未变化。

原因：GitHub Pages 默认部署分支可能不是 main，或存在缓存延迟。

✅ 解决：前往仓库 Settings -> Pages，确认部署分支为 main，保存后等待片刻即可。

❌ 坑三：Vercel 重复构建

现象：导入 Hexo 项目后 Vercel 再次执行 hexo g，导致构建失败或超时。

原因：Vercel 检测到 Hexo 项目后自动添加了构建命令，与已生成的静态文件冲突。

✅ 解决：在 Vercel 项目设置中找到 Build Command，启用 Override 并留空，让 Vercel 直接部署现有静态文件。

❌ 坑四：本地预览与线上效果不一致

现象：hexo s 本地完美，部署后样式丢失或特效失效。

原因：缓存问题（浏览器/CDN/Service Worker）是主要原因。另外，Butterfly 开启 PJAX 无刷新跳转后，自定义 JS 不会在页面切换时重新执行。

✅ 解决：

  1. 清理缓存：hexo clean + 浏览器无痕模式 + 强刷（Ctrl + F5）。如有 PWA，需清除 Service Worker。
  2. PJAX 适配：将自定义 JS 包裹在 document.addEventListener('pjax:complete', function() {...}) 中，确保每次切页重新触发。

最终成果与碎碎念

折腾一圈，看着博客从简陋页面一点点加上昼夜切换、音乐播放、评论留言，变成现在的顺眼模样，所有头秃时刻都值了。

建站最大的门槛其实是**"开始动手"**。很多时候我们怕报错、怕踩坑，但正是这些折腾，让你摸清了前端、Git、部署的皮毛。博客搭好只是第一步，希望未来我们都能在这块小小的自留地里，坚持记录真实的生活与思考。

折腾不息，共勉！
