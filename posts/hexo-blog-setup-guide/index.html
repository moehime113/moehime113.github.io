Hexo 博客建站实录

2024/06/15八宝箱

🌟 初衷：为什么想做一个博客？

一直想有块完全属于自己的“互联网自留地”，不受平台限制，能自由定义细节。比了一圈，最后敲定了轻快高颜值的组合：Hexo + Butterfly 主题。

下面聊聊我从零搭建、魔改到上线的真实体验，顺便盘点那些让人抓狂的坑，帮大家少走弯路。

🛠️ 折腾第一步：让博客先跑起来

建站就像搭积木，环境弄好，剩下就是顺水推舟。

1. 搞定基础环境

备好两样工具：

  • Git & GitHub：做版本控制和网页托管。
  • Node.js：Hexo 的运行基础，官网下个稳定版装上就行。

2. 初始化 Hexo 与主题

打开终端，敲几行命令搭好骨架：

# 安装 hexo-cli，初始化博客目录
npm install -g hexo-cli
mkdir your_blog && cd your_blog
hexo init && npm install

接着装上 Butterfly 主题：

npm install hexo-theme-butterfly

修改根目录 _config.yml，把 theme 设为 butterfly。终端跑一句 hexo s，打开 http://localhost:4000——看到雏形加载出来的那一刻，还是挺有成就感的！

🚀 部署上线：让世界看到你的博客

本地自嗨不够，得把它挂到网上。

1. 部署到 GitHub Pages

装好 Git 部署插件 hexo-deployer-git，在配置文件填上 GitHub 仓库地址和分支（推荐 main）。然后执行经典的“三连”命令推送到云端：

hexo clean && hexo g && hexo d

2. 进阶玩法：Vercel 与评论系统部署

如果想要国内访问更快，强烈推荐用 Vercel 托管静态页面。 另外，我后来还折腾了 Twikoo 评论系统，同样用 Vercel 部署云函数。免费好用，不用操心服务器，完美契合静态博客。

🔨 魔改与避坑指南：那些折磨过我的 Bug

跟着教程走一帆风顺，自己动手加特效（比如轮播图、Aplayer 音乐）时，玄学问题就来了。盘点几个大坑：

❌ 坑一：静态资源加载失败（CDN 之痛）

现象：本地跑得好好的，一部署线上控制台就一片红 ERR_ABORTED，排版错乱、图标变方块。 原因：老教程推荐的国内/小众 CDN（如 tianli0.top、bootcdn）很多已停运，或在 GitHub/Vercel 环境被拦截。 ✅ 解决：果断弃用！在 _config.butterfly.yml 里，把第三方资源统统换成国际大厂 CDN，比如 unpkg.com（最稳妥）或 cdn.jsdelivr.net/npm/。

❌ 坑二：GitHub Actions 自动部署页面不更新

现象：Workflow 自动构建并推送到 main 分支显示成功，但网页死活不更新。 原因：GitHub Pages 默认可能还在傻傻读取旧的 gh-pages 分支。 ✅ 解决：去 GitHub 仓库的 Settings -> Pages，把部署分支手动改为 main，保存稍等片刻即可。

❌ 坑三：Vercel 部署重复构建报错

现象：把跑过 Actions 的 main 分支导入 Vercel，构建卡住甚至报错。 原因：Actions 已生成纯静态文件，但 Vercel 发现是 Hexo 项目，又自作多情帮你跑了一遍构建命令。 ✅ 解决：在 Vercel 项目设置里找到 Build Command，打开 Override 并留空。让 Vercel 把它当纯静态站点直接部署，清爽快速。

❌ 坑四：本地预览和线上效果不一致（最恶心的玄学）

现象：hexo s 本地看一切完美，推送到线上后，发现样式没变、自定义的特效出不来，甚至页面错乱。 原因：这通常和 缓存（浏览器/CDN/PWA 服务端缓存） 或 PJAX 异步加载 有关。本地是即时响应的，而线上部署后，浏览器或 JS 服务端（Service Worker）会强缓存旧文件。另外，如果加了自定义 JS 特效，由于 Butterfly 开启了 PJAX 无刷新跳转，换页时 JS 可能不会重新执行。 ✅ 解决：

  1. 清缓存：遇事不决先 hexo clean，线上调试务必用浏览器的 无痕模式 或强刷（Ctrl + F5）。如果开了 PWA，记得去控制台清理 Service Worker。
  2. PJAX 适配：如果是自定义 JS 特效失效，要把 JS 代码包在 PJAX 重载事件里（如 document.addEventListener('pjax:complete', function() {...})），确保每次切页都能重新触发。

💡 最终成果与碎碎念

折腾一圈，看着博客从简陋页面一点点加上昼夜切换、音乐播放、评论留言，变成现在的顺眼模样，所有头秃时刻都值了。

建站最大的门槛其实是**“开始动手”**。很多时候我们怕报错、怕踩坑，但正是这些折腾，让你摸清了前端、Git、部署的皮毛。博客搭好只是第一步，希望未来我们都能在这块小小的自留地里，坚持记录真实的生活与思考。

折腾不息，共勉！
